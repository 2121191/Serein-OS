下面是 xv6-user 目录中**用户程序与库**的清单（中文简短说明），帮助你快速了解每个程序的功能与用途：

- init / _init (init.c / initcode) — 系统启动的第一个用户进程，负责启动 shell 等初始化工作。  
- sh / _sh (sh.c) — XV6 的 shell，解析用户命令并启动其他用户程序（支持重定向、管道等）。  
- cat / _cat (cat.c) — 输出文件内容或从标准输入复制到标准输出（实现类似 Unix cat）。  
- echo / _echo (echo.c) — 打印传入的参数（类似 shell echo）。  
- ls / _ls (ls.c) — 列出目录内容。  
- mkdir / _mkdir (mkdir.c) — 创建目录。  
- rm / _rm (rm.c) — 删除文件。  
- mv / _mv (mv.c) — 移动/重命名文件（实现为 link+unlink 或类似逻辑）。  
- ln (ln.c) — 创建硬链接（通常被注释掉或非默认启用）。  
- kill / _kill (kill.c) — 发送信号（在 xv6 中用于终止进程）。  
- sleep / _sleep (sleep.c) — 让进程睡眠指定时长（测试时钟中断/调度）。  
- wc / _wc (wc.c) — 统计文件行/字/字节数（类似 Unix wc）。  
- grep / _grep (grep.c) — 在输入中搜索匹配模式。  
- find / _find (find.c) — 在目录树中查找文件名。  
- xargs / _xargs (xargs.c) — 将标准输入或参数用于执行命令（测试 shell 与参数传递）。  
- forktest (forktest.c) — 用于压力测试 fork 行为（通常小巧以最大化进程数测试）。  
- test / _test (test.c) — 一些示例或测试用例程序（具体实现视仓库）。  
- usetests / _usertests (usertests.c) — 一组自动化用户态测试集合（用作回归）。  
- strace / _strace (strace.c) — 跟踪系统调用（打印调用与返回，便于调试）。  
- stressfs (stressfs.c) — 文件系统压力测试工具（通常用于发现并发或一致性错误）。  
- myprog / _myprog (myprog.c) — 你刚添加的测试程序（演示文件读写、fork/wait）。  
- printf / ulib / umalloc (printf.c, ulib.c, umalloc.c) — 用户态库实现（printf、基本库函数、简单内存分配等）。  
- usys.S / usys.pl — 生成用户态的系统调用封装（将系统调用导出为用户函数）。  
- user.h — 用户态程序的头文件（包含 syscalls 和标准接口常量/声明）。  
- xargstest.sh — 一个脚本测试 xargs 功能（集成测试示例）。  
- zombie.c — 用来测试僵尸进程/wait 行为的示例。
